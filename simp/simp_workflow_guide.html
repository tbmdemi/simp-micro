🔍 Xác định bài toán Bước đầu

Trước khi chạy, bạn cần xác định rõ:

  • Mục tiêu: Bạn muốn tối ưu độ cứng cắt (E1122) hay kết hợp với ràng buộc độ cứng dọc?
  • Hạt mầm: Dạng hình học ban đầu nào phù hợp? (tròn, vuông, đồng hồ cát, lục giác...)
  • Kích thước lưới: Lưới càng mịn → kết quả chi tiết hơn nhưng chậm hơn.
  • Tỉ lệ thể tích: Bao nhiêu phần trăm ô đơn vị được phép chứa vật liệu?

⚙️ Cấu hình tham số config.py

Tất cả tham số tập trung trong simp/config.py — class SimpConfig. Bạn có thể:

  • Qua CLI: Python -m simp.main --nelx 60 --volfrac 0.35
  • Qua code: Tạo SimpConfig(nelx=60, volfrac=0.35) trong script Python
  • Qua notebook: Import config và chỉnh sửa trực tiếp
💡
Mẹo: Dùng --list-Hạt Giống để xem tất cả hạt mầm có sẵn trước khi chọn.
4

🏃 Chạy tối ưu hóa runner.py

Vòng lặp chính được điều phối bởi run_simp() trong runner.py. Quy trình mỗi vòng lặp:

  1. FEM → solve_fe(): Lắp ráp K, giải hệ PBC
  2. Đồng nhất hóa → compute_homogenized_tensor(): Tính Q (3×3)
  3. Mục tiêu → compute_first/second_objective(): Tính c và dc
  4. Lọc → H @ dc: Làm mượt gradient, tránh checkerboard
  5. OC → optimize_oc(): Cập nhật mật độ x theo OC
  6. Hội tụ → check_convergence(): Kiểm tra change < tol
⚠️
Quan trọng: Lần chạy đầu tiên luôn dùng --verbose để theo dõi tiến trình. Nếu thấy NaN trong objective → dừng ngay và kiểm tra tham số.
5

📊 Phân tích kết quả output

Sau khi chạy xong, dữ liệu được lưu trong thư mục output (mặc định: test_/):

  • iteration_data.csv — Lịch sử hội tụ (objective, thể tích, change)
  • iteration_*.png — Ảnh chụp cấu trúc tại từng bước
  • Phân tích hội tụ: Dùng notebooks/01_convergent_analysis.ipynb
  • Phân tích ảnh: Dùng notebooks/02_image_quality_analysis.ipynb

🔄 Điều chỉnh & lặp lại tuning

Dựa trên kết quả phân tích, bạn có thể:

  • Tăng/giảm volfrac nếu kết quả quá đặc hoặc quá rỗng
  • Thay đổi seed nếu cấu trúc hội tụ không như mong muốn
  • Điều chỉnh rmin (bán kính lọc) để kiểm soát độ phức tạp
  • Thử objective khác (first_obj ↔ second_obj) nếu cần ràng buộc thêm

Chạy từ đầu — Step-by-Step

Hướng dẫn chi tiết cho người mới bắt đầu: từ clone repo đến có kết quả

1. Clone & cài đặt

git clone && cd Input_SIMP_Analyst && pip install -r requirements.txt

2. Chạy thử nghiệm đầu tiên

Python -m simp.main --seed circle --nelx 60 --nely 60 --volfrac 0.4 --verbose

3. Xem kết quả

Mở thư mục test_circle/ — xem file iteration_data.csv và các ảnh PNG.

4. Phân tích hội tụ

Mở notebooks/01_convergent_analysis.ipynb trong Jupyter để vẽ biểu đồ hội tụ.

5. Thử với seed khác

Python -m simp.main --seed hourglass --nelx 60 --nely 60 --volfrac 0.35

6. Thử objective thứ hai

Python -m simp.main --seed circle --objective second --nelx 60 --nely 60

✅
Kết quả mong đợi: Sau 30–50 vòng lặp, objective sẽ hội tụ về một giá trị ổn định. Cấu trúc trong ảnh PNG sẽ hiện rõ hình dạng tối ưu với các thanh/vùng vật liệu phân bố hợp lý.

Cách thay đổi & tuỳ chỉnh

Các kịch bản thường gặp khi bạn muốn can thiệp sâu vào thuật toán

🔄 Đổi hạt mầm

Mỗi hạt mầm là một file trong simp/Hạt Giống/. Bạn có thể:

  • Dùng CLI: --seed four_circle
  • Thêm seed mới: tạo file simp/Hạt Giống/my_seed.py với hàm generate(config) → x
  • Đăng ký seed mới trong simp/Hạt Giống/__init__.py

🎯 Đổi hàm mục tiêu

Hai objective có sẵn trong simp/Mục Tiêu/:

  • first_obj: Chỉ tối ưu E1122 (độ cứng cắt)
  • second_obj: Tối ưu E1122 với ràng buộc E1111 ≥ threshold
  • Bạn có thể tự viết objective mới theo interface tương tự

📐 Đổi kích thước lưới

Thay đổi nelx và nely:

  • Lưới 30×30: chạy nhanh (~5s), kết quả thô
  • Lưới 60×60: cân bằng (~30s)
  • Lưới 120×120: chi tiết (~3 phút)
  • Lưới 240×240: rất chậm (~20 phút), chỉ dùng khi cần độ chính xác cao

⚡ Đổi hệ số phạt (penal)

penal kiểm soát độ "sắc nét" của kết quả nhị phân:

  • penal = 1: Kết quả mờ, nhiều xám (density không về 0/1)
  • penal = 3: Mặc định, cân bằng tốt
  • penal = 5: Kết quả rất sắc, nhưng dễ bị local optima

🔍 Đổi bán kính lọc (rmin)

rmin ảnh hưởng đến độ phức tạp hình học:

  • rmin = 1.0: Nhiều chi tiết nhỏ, có checkerboard
  • rmin = 1.5: Mặc định, an toàn
  • rmin = 3.0: Cấu trúc đơn giản, mượt

📊 Đổi tỉ lệ thể tích (volfrac)

Kiểm soát lượng vật liệu trong ô đơn vị:

  • volfrac = 0.3: Kết cấu nhẹ, nhiều lỗ rỗng
  • volfrac = 0.5: Cân bằng
  • volfrac = 0.7: Kết cấu đặc, ít lỗ rỗng
  • Lưu ý: volfrac quá thấp có thể gây mất ổn định số

Lưu ý quan trọng khi thao tác

Những điều cần ghi nhớ để tránh mất thời gian debug

⚠️ NaN trong objective

Nguyên nhân thường gặp:

  • volfrac quá thấp (< 0.2) → ma trận K suy biến
  • penal quá cao (> 6) → gradient không ổn định
  • Lưới quá thô (< 20×20) → không đủ bậc tự do
  • Emin quá nhỏ → ma trận K gần singular
🔧
Fix: Tăng volfrac lên 0.3, giảm penal xuống 3, kiểm tra Emin (nên ≥ 1e-9).

🔄 Không hội tụ

Objective dao động liên tục, không ổn định:

  • move quá lớn (> 0.3) → giảm xuống 0.15
  • tol quá nhỏ (< 0.001) → tăng lên 0.01
  • rmin quá nhỏ (< 1.0) → tăng lên 1.5
  • max_iter quá ít (< 50) → tăng lên 200

🧩 Checkerboard pattern

Hiện tượng các ô đen/trắng xen kẽ nhau:

  • Nguyên nhân: rmin quá nhỏ hoặc không dùng Lọc
  • Fix: Tăng rmin lên ≥ 1.5
  • Kiểm tra Lọc có được áp dụng không trong runner.py

🐌 Chạy quá chậm

Thời gian chạy quá lâu:

  • Giảm nelx/nely — 60×60 là đủ cho hầu hết trường hợp
  • Giảm max_iter — nếu hội tụ sớm, không cần chạy hết
  • Tăng save_every — giảm số lần ghi ảnh
  • Tắt verbose — I/O cũng tốn thời gian

📁 Output bị ghi đè

Chạy lại cùng seed → mất dữ liệu cũ:

  • Luôn đặt output_dir khác nhau cho mỗi lần chạy
  • Hoặc dùng timestamp: f"test_{seed}_{datetime.now():%Y%m%d_%H%M}"
  • Backup dữ liệu quan trọng trước khi chạy lại

🔢 Kết quả không tái lập

Chạy lại cho kết quả khác:

  • Kiểm tra random seed trong code
  • Nếu có phần ngẫu nhiên, set np.random.seed(42)
  • Đảm bảo cùng version thư viện (NumPy, Scipy)

Debug & Xử lý lỗi

Các lỗi thường gặp và cách khắc phục nhanh

❌ "Singular matrix" / "LinAlgError" +

Nguyên nhân: Ma trận độ cứng K không khả nghịch — thường do quá nhiều ô rỗng (Emin quá nhỏ) hoặc volfrac quá thấp.

Cách fix:

  1. Tăng Emin lên 1e-6 hoặc 1e-5
  2. Tăng volfrac lên ≥ 0.3
  3. Kiểm tra điều kiện biên PBC trong simp/core/pbc.py
  4. Nếu vẫn lỗi, thử dùng lưới lớn hơn (nelx ≥ 40)
❌ "ModuleNotFoundError: No module named 'simp'" +

Nguyên nhân: Chưa cài đặt package dạng editable.

Cách fix:

{{CODE_BLOCK_2}}
❌ "ValueError: operands could not be broadcast together" +

Nguyên nhân: Kích thước mảng không khớp — thường do thay đổi nelx/nely mà không cập nhật seed tương ứng.

Cách fix:

  • Kiểm tra seed function có dùng config.nelx và config.nely không
  • Một số seed cứng (hardcoded) kích thước — cần sửa lại
  • Dùng --list-Hạt Giống để xem seed nào linh hoạt
❌ Kết quả toàn màu xám (không về 0/1) +

Nguyên nhân: Hệ số phạt chưa đủ mạnh hoặc chưa đủ số vòng lặp.

Cách fix:

  • Tăng penal lên 4 hoặc 5
  • Tăng max_iter lên 300
  • Kiểm tra OC update có hoạt động đúng không
  • Thử giảm move xuống 0.1 để thay đổi chậm hơn
💡
Mẹo debug tổng quát: Luôn chạy với --verbose ở lần đầu. Log sẽ hiển thị objective, change, và thể tích sau mỗi vòng lặp — giúp bạn phát hiện sớm các bất thường.

Đề xuất cải tiến & Phát triển thêm

Những phần có thể thêm vào để SIMP mạnh mẽ và linh hoạt hơn

📈 CAO Adaptive mesh refinement (AMR)

Tự động làm mịn lưới ở những vùng có gradient mật độ cao (biên giữa vật liệu và lỗ rỗng). Giúp:

  • Tăng độ chính xác mà không tăng tổng số ô lưới
  • Tập trung tài nguyên tính toán vào vùng quan trọng
  • Có thể dùng Scipy.interpolate để map kết quả giữa các lưới

🧠 CAO Multi-objective Tối Ưu Hóa (MOO)

Thay vì chỉ tối ưu một mục tiêu, cho phép tối ưu đồng thời nhiều mục tiêu:

  • Ví dụ: tối đa E1122 + tối thiểu khối lượng + ràng buộc độ bền
  • Có thể dùng phương pháp weighted sum hoặc Pareto front
  • Thêm class MultiObjective trong simp/Mục Tiêu/

🔄 CAO Restart từ checkpoint

Cho phép lưu trạng thái hiện tại và tiếp tục chạy sau:

  • Lưu x (mật độ), iteration, objective_history vào file .npy hoặc .h5
  • Thêm flag --restart checkpoint.npy
  • Rất hữu ích khi chạy lưới lớn (120×120+) bị gián đoạn

📊 TRUNG BÌNH Sensitivity analysis

Phân tích độ nhạy của kết quả với từng tham số đầu vào:

  • Dùng thư viện SALib để chạy Morris / Sobol analysis
  • Xác định tham số nào ảnh hưởng nhất đến objective
  • Giúp tập trung tuning vào đúng tham số quan trọng

🤖 TRUNG BÌNH GPU acceleration (CuPy / JAX)

Tận dụng GPU để tăng tốc FEM solver:

  • Thay NumPy bằng cupy hoặc jax.NumPy
  • Giảm thời gian chạy từ phút xuống giây cho lưới lớn
  • Cần có NVIDIA GPU + CUDA toolkit

📝 TRUNG BÌNH Automatic hyperparameter tuning

Tự động tìm tham số tối ưu:

  • Dùng Optuna hoặc Scipy.optimize
  • Tối ưu đồng thời volfrac, penal, rmin, move
  • Kết hợp với early stopping để tiết kiệm thời gian

📦 THẤP Export kết quả sang CAD/STL

Chuyển kết quả tối ưu thành file CAD để in 3D hoặc mô phỏng:

  • Dùng Scipy.ndimage để làm mượt biên
  • Export sang STL qua NumPy-stl
  • Hoặc dùng trimesh để xử lý mesh

🌐 THẤP Web dashboard real-time

Xem tiến trình tối ưu hóa trực tiếp trên web:

  • Dùng Flask + Socket.IO để stream dữ liệu
  • Hiển thị biểu đồ hội tụ và ảnh cấu trúc real-time
  • Cho phép điều chỉnh tham số giữa chừng

🧪 THẤP Unit test & CI/CD

Đảm bảo code luôn chạy đúng sau mỗi lần sửa:

  • Viết thêm test cho từng module (đã có tests/)
  • Thêm GitHub Actions để tự động chạy test
  • Kiểm tra coverage để biết phần nào chưa được test
